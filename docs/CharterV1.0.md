## Core Institutions

### 1 — The Compilation Contract

No idea is "real" in RG² until it compiles into a bundle.

A valid bundle MUST contain:

- `CLAIM.md` (≤3 sentences; bounded; falsifiable)
- `OPERATIONALIZE.md` (observables + estimators + windows)
- `PREREG.yaml` (locked params, seeds, stopping rules, forbidden deviations)
- `NULLS.md` (≥2 numeric rejection thresholds)

If it doesn't compile, it's not yet a claim.

---

### 2 — Hard Gates

Hard gates are non-negotiable and enforced automatically.

Minimum gates:

- **G1: Numeric null completeness** — `NULLS.md` contains ≥2 numeric thresholds that reject the claim.
- **G2: Bounded claim** — claim is short, specific, and scope-limited.
- **G3: Operational definitions** — every key term maps to a metric or procedure.
- **G4: Prereg rigidity** — thresholds and sweep ranges can't be tuned post hoc.

Failing a gate triggers RESTART, not debate.

---

### 3 — Coherence Controller

RG² measures "constraint health" during compilation and uses it to decide:

- **CONVERGE** (freeze bundle)
- **DIVERSIFY** (inject alternative skeptics/models)
- **RESTART** (recompile with explicit defect list)

Premature convergence is a failure mode.
If dissent collapses early while health is low, the system must diversify.

---

### 4 — Evidence Bar

Evidence is tracked per bundle, not per vibe.

- **E5:** Reproducible code + data + tests + full trace + independent rerun
- **E4:** Methods-grade artifact + robustness sweeps + negative controls + prereg compliance
- **E3:** Reproducible sim/experiment with clear null outcomes
- **E2:** Domain-common knowledge (explicitly labeled)
- **E1:** Speculation (must include a minimal decisive test)

No claim may be described as E3+ without a bundle and run artifacts.

---

### 5 — Modeling Protocol (Executable)

Prefer symbolic/parametric first.

Every model spec must declare:

- Variables, units, invariants
- Symmetries and conserved quantities (if claimed)
- Measurement windows and estimators
- Failure modes and null thresholds
- Seeds and reproducibility knobs

Every runnable model must ship with:

- Parameter scan script
- Sanity checks
- Seed manifest
- Regression tests for metrics

---

### 6 — Prototype Protocol (Instrumented)

No prototype is "built" until it is measurable.

Every prototype spec includes:

- BOM + wiring/signal path
- Calibration plan
- Data schema + sampling rate
- Pass/fail thresholds
- Negative controls (placebo, sham, decoy channels)

---

### 7 — Integrity Rules

- Separate metaphor from math.
- Label speculation.
- No post-hoc threshold tuning.
- If a claim hinges on unknowns: propose the smallest decisive experiment.
- Preserve failures with the same seriousness as wins.

Failures are artifacts, not embarrassments.
